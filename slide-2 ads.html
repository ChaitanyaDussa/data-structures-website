concept

Algorithm analysis evaluates the efficiency of an algorithm in terms of time and space complexity, helping to predict its performance. It identifies the resources an algorithm needs as input size grows, crucial for optimizing code.

  • Space and Time Complexity part-1
  • Space and Time Complexity part-2
  • Space and Time Complexity part-3

concept

Space complexity measures the total memory an algorithm requires relative to input size, including auxiliary and temporary space. It's calculated by identifying memory allocated for variables, data structures, and function calls.
Time complexity estimates the total time an algorithm takes to run as input size grows. Calculations are based on counting operations to determine growth rates; examples include O(1) for constant time or O(n²) for quadratic time in nested loops.

  • Asymptotic Notations

concept

Asymptotic notations describe algorithm efficiency at large input sizes: Big O for worst-case (upper bound), Theta (Θ) for average-case (tight bound), and Omega (Ω) for best-case (lower bound). Little o and ω give non-tight upper and lower bounds. Growth functions (polynomial, exponential, logarithmic) help compare algorithm scalability. Recursive algorithms are analyzed with recurrence relations and the Master Theorem to predict performance.

  • AVL Tree Rotations

concept

AVL trees are self-balancing binary search trees where the height difference (balance factor) between left and right subtrees is at most one, ensuring efficient operations. To maintain balance, AVL trees use single (left/right) and double (left-right/right-left) rotations, preserving quick search, insertion, and deletion

  • AVL Tree Deletion

concept

Creating an AVL tree starts with an empty tree, where nodes are added with balance maintained through rotations during insertion and deletion. Insertion and deletion may trigger rebalancing using single or double rotations to address imbalance cases, ensuring efficient operations.

->Applications of AVL Trees

concept

AVL trees are widely used in applications needing fast data retrieval, such as database indexing, where balanced trees speed up searches. They’re also applied in memory management and optimizing search operations within large datasets.

->Applications of B-Trees

concept

B-trees are self-balancing multi-way search trees optimized for systems that read and write large blocks of data, such as databases and file systems. They maintain efficient insertion, deletion, and search operations through structured node splitting and balancing, widely applied in database indexing, disk storage, and multi-level indexing.

  • Heap Trees (Priority Queues)

concept

Heap Trees (Priority Queues) – Heaps, including min-heaps and max-heaps, are binary trees used to manage prioritized data efficiently, with applications in sorting, scheduling, and optimizing algorithms like Dijkstra's for shortest paths.

  • Graph Representation
  • BFS and DFS Graph Traversals

concept

Graphs – Graphs represent networks with nodes and edges, enabling analyses like search (DFS, BFS), connected components, and biconnected components, which are critical in network connectivity, dependency resolution, and traversal applications.

  • Divide and Conquer Algorithm - General Method
  • Quick Sort Algorithm
  • Merge Sort Algorithm
  • Strassen's Matrix Multiplication
  • convex hull algorithm

concept

Divide and Conquer – This strategy divides a problem into smaller subproblems, solving and combining them, and is fundamental in algorithms like Quick Sort, Merge Sort, Strassen’s matrix multiplication, and geometric problems like finding the Convex Hull.

  • General Method of Greedy Algorithms

concept

General Method: Greedy algorithms build up a solution piece by piece, choosing the option that offers the immediate benefit. They are efficient for problems with optimal substructure but don't always guarantee global optimality.

  • Job Sequencing with Deadlines

concept

Job Sequencing with Deadlines: In this problem, jobs with deadlines and profits are scheduled to maximize profit. The greedy approach selects jobs based on their profits, aiming to maximize earnings within the given deadlines.

  • Single Source Shortest Paths

concept

Single Source Shortest Paths: This problem aims to find the shortest paths from a single source to all other vertices in a graph. Greedy algorithms like Dijkstra's are effective when all edge weights are non-negative.

  • Knapsack Problem using greedy method

concept

Knapsack Problem: In the fractional knapsack variant, the goal is to maximize value by taking fractional quantities of items that fit within a weight limit. The greedy approach selects items based on the highest value-to-weight ratio.

  • Minimum Cost Spanning Trees using Greedy method

concept

Minimum Cost Spanning Trees: This involves finding a subset of edges in a graph that connects all vertices with the minimum possible total edge weight. Greedy algorithms like Prim’s and Kruskal’s are commonly used for constructing these trees.

  • General Method of Dynamic Programming

concept

General Method: Dynamic programming (DP) is used for problems with overlapping subproblems and optimal substructure. DP breaks problems into subproblems, solves each once, and stores their solutions to avoid redundant computations.

  • 0/1 Knapsack Problem using dynamic programming

concept

0/1 Knapsack: In this variation, items cannot be divided, and each item is either included or excluded to maximize value without exceeding weight capacity. DP is used to consider each item and weight limit combination.

  • All-Pairs Shortest Paths using dynamic programming

concept

All Pairs Shortest Paths: This problem finds the shortest paths between all pairs of vertices in a graph. The Floyd-Warshall algorithm is a DP approach that iteratively considers each vertex as an intermediate step between pairs.

  • Single Source Shortest Paths

concept

Single Source Shortest Paths – General Weights (Bellman-Ford Algorithm): This algorithm calculates the shortest paths from a single source in graphs with negative weights. It’s a DP approach that iteratively relaxes edges and can detect negative-weight cycles.

  • Optimal Binary Search Trees (OBST)

concept

Optimal Binary Search Trees: This problem involves constructing a binary search tree that minimizes search costs given a set of keys with access probabilities. DP helps to find the structure of the tree for minimum expected search time.

  • Travelling Salesperson Problem (TSP) using dynamic programming

concept

Travelling Salesperson Problem (TSP): TSP seeks the minimum-cost route that visits each city exactly once and returns to the starting city. DP approaches like Held-Karp use subproblem results to find optimal paths for progressively larger subsets of cities.

  • String Editing (Edit Distance)

concept

String Editing: The string editing (or edit distance) problem finds the minimum operations needed to transform one string into another. DP provides a table-based solution by calculating edit costs for each substring alignment.

  • 0/1 Knapsack Problem using backtracking

concept

0/1 Knapsack Problem: Similar to the DP version, this variation involves selecting items to maximize value within a weight limit. Backtracking explores each item inclusion/exclusion option, pruning paths when limits are exceeded.

  • Graph Coloring Problem - Backtracking

concept

Graph Coloring: In this problem, vertices of a graph are colored using a minimum number of colors such that no adjacent vertices share the same color. Backtracking assigns colors one by one, backtracking on conflicts.

  • Sum Of Subsets Problem - Backtracking

concept

Sum of Subsets Problem: Given a set of integers and a target sum, the goal is to find all subsets that add up to the target. Backtracking explores each subset combination and prunes paths that exceed the target.

  • N-Queens Problem
  • 8-Queens Problem

concept

8-Queens Problem: This classic problem places 8 queens on an 8x8 chessboard such that no two queens threaten each other. Backtracking tries to place queens row by row, undoing placements if they lead to conflicts.

  • Branch and Bound: The General Method

concept

General Method: Branch and Bound is used for optimization problems. It divides the problem into smaller subproblems (branching) and uses bounds to eliminate suboptimal solutions, focusing on paths that might yield the best solution.

  • Branch and Bound: 0/1 Knapsack Problem

concept

0/1 Knapsack Problem: In Branch and Bound, the 0/1 knapsack problem is solved by creating a solution tree where nodes represent item inclusion/exclusion. Bounds help prune subproblems that cannot improve the optimal solution.

  • Traveling Salesperson Problem (TSP) using brancha and bound

concept

Travelling Salesperson Problem (TSP): The goal is to find the minimum-cost route that visits each city once and returns to the start. Branch and Bound generates subproblems (partial tours) and prunes based on lower-bound costs to find the optimal tour.

  • NP Hard and NP Complete Problems

concept

Basic Concepts: NP-Hard problems are as hard as the hardest problems in NP (nondeterministic polynomial time), meaning that there’s no known polynomial-time solution. NP-Complete problems are both in NP and NP-Hard, making them some of the most challenging problems to solve efficiently.

  • Cook's Theorem

concept

Cook’s Theorem: This foundational theorem, proposed by Stephen Cook, states that the Boolean satisfiability problem (SAT) is NP-Complete. This implies that if SAT can be solved in polynomial time, all NP problems can also be solved in polynomial time, thus proving P=NP.

  • NP-Hard Graph Problem - Clique Decision Problem

concept

Clique Decision Problem (CDP): In this problem, the goal is to determine if a graph contains a clique (a subset of vertices that are all adjacent) of a given size. It’s NP-Hard because finding such cliques requires checking combinations of vertices, which is computationally intensive.

  • Chromatic number of a Graph

concept

Chromatic Number Decision Problem (CNDP): This problem asks whether a graph can be colored with a given number of colors such that no two adjacent vertices share the same color. It is NP-Hard because of the complexity of exploring all possible color assignments.

  • Traveling Salesperson Problem

concept

Traveling Salesperson Decision Problem (TSP): The decision version of TSP asks whether there exists a tour that visits each city exactly once and returns to the start with a cost below a specified threshold. It’s NP-Hard because the number of possible routes grows factorially with the number of cities.

  • Scheduling Identical Processors

concept

Scheduling Identical Processors: This problem involves assigning tasks to identical processors to minimize the overall completion time (makespan). It’s NP-Hard because of the difficulty in optimally distributing tasks among processors within minimal time.

  • Job Shop Scheduling

concept

Job Shop Scheduling: In this scheduling problem, a set of jobs with specific task orders must be processed on multiple machines. The objective is often to minimize completion time, making it NP-Hard due to the complex dependency and resource allocation constraints.
